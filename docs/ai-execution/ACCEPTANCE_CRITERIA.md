# Acceptance Criteria

## 全局验收

- `mvn test` 必须通过。
- 后端启动时，缺少 LLM API Key 不能导致应用启动失败。
- 所有新增表结构必须有 Flyway 迁移。
- 所有接口返回 `Result<T>`。
- 需要鉴权的接口必须校验用户身份和资源归属。
- 任何异步任务必须能通过 `/api/v1/tasks/{taskId}/progress` 查询进度。

## 账号体系

- 注册后自动登录，但 `isEmailVerified=false` 时 AI 功能不可用。
- 邮箱验证 token 24 小时有效，重发接口 60 秒限流。
- 邮件模板必须包含邮箱验证、密码重置、安全告警三类。
- 登录连续失败 5 次锁定 15 分钟。
- Refresh Token 30 天有效，刷新时轮换，旧 Token 立即失效。
- 忘记密码接口不得泄露邮箱是否存在；不存在邮箱返回成功式响应但不发送重置邮件。
- 密码重置 token 24 小时有效且只能使用一次。
- 密码重置成功后所有设备退出，旧 Refresh Token 立即失效。
- 账号注销为软删除，30 天内可撤销。
- 首次登录必须选择身份类型，身份只影响入口顺序和引导，不限制功能权限。

## 编辑器与章节

- 任意时刻前端编辑器只加载当前章节。
- 章节保存必须记录 `word_count`。
- 每次 AI 大改、润色接受、全书替换前必须自动创建快照。
- 快照恢复前必须先保存当前状态为新快照。

## 搜索替换

- 搜索由后端执行，前端不得把全卷加载进浏览器搜索。
- 搜索支持大小写敏感和整词匹配选项。
- 替换前创建 `auto_before_replace` 快照。
- 命中角色名时，必须提供同步更新角色档案选项。

## 写作统计

- 统计采用差量采集：章节保存时比较旧字数和新字数。
- 支持今日、7 日、30 日、月度、连续日更、热力图、作品进度。
- 每次章节保存后统计可被下一次查询读到。

## AI 续写

- SSE 必须有首字 8 秒超时事件。
- 用户中断 SSE 后，后端尽量取消 LLM 调用，未完成内容不扣最终配额。
- AI 内容进入前端后保持未接受态，有绿色左侧标识。
- 接受 AI 内容后刷新配额并创建版本快照。

## 上下文锚点

- 锁定角色必须优先注入上下文，不受普通预算裁剪。
- 历史摘要、近期原文、世界观、角色档案按 Token 预算组装。
- 章节完成、AI 续写接受、手动新增 1000 字以上应触发锚点更新。

## 润色与一致性审查

- 基础校正返回逐条修改建议。
- 进阶润色支持全部接受/全部拒绝。
- 风格重塑和专属文风必须明确消耗预估。
- 一致性审查按套餐限制单次字数和月次数。
- 审查报告条目支持 `open/handled/ignored`。

## 剧本改编

- 改编任务必须异步上报进度。
- 剧本工作台返回场景目录、原文、剧本、AI 建议所需数据。
- 场景编辑必须使用乐观锁，版本冲突返回 HTTP 409 和最新版本。
- 导出任务完成后返回腾讯云 COS 预签名 URL。

## 前端交互

- UI 使用 Naive UI。
- TipTap SSE token 写入必须 RAF 批量处理。
- 配额 Tooltip 数据最多允许 60 秒延迟，并提示“配额用量每次操作完成后更新”。
- 全文搜索横条 300ms debounce，Esc 关闭。
- 移动端不加载复杂剧本工作台，显示 PC 使用提示。
- 命令面板必须支持 Cmd/Ctrl+K、键盘导航、快捷键帮助入口。
- Toast 必须支持右下角堆叠、自动消失、手动关闭和移动端顶部展示。
- 异常状态必须覆盖服务不可用、结果不可用、配额不足、内容被替换四级。
- 渐进式引导只在首次触发场景展示，用户关闭后不重复打扰。
- 快捷键参考面板必须覆盖导航、章节跳转、AI 操作、视图切换。
- 前端必须通过代码分割避免移动端下载剧本工作台、diff 等重组件。

## 商业化、团队与合规

- 免费体验额度、月度订阅配额、字数包必须区分来源但对用户合并展示。
- 操作前必须展示配额消耗预估，配额不足时数字变红并提示风险。
- 团队版必须支持成员邀请、角色管理、共享配额和统一用量账单。
- 项目协作者权限必须校验，非协作者不可访问团队作品。
- 导出剧本必须添加“AI 辅助生成”页脚标注。
- 作品默认私有，分享链接如后续实现必须有有效期，默认 24 小时。
- 用户可随时关闭 AI 训练授权，关闭后立即生效。
- 内容安全替换必须透明展示原因，并提供误报申诉入口。
