1. 产品概述

1.1 产品定位

文枢（wenshu） 是一款专为长篇创作者打造的一站式智能创作平台。以 "小说转剧本" 为核心差异化功能，辅助 "可控长篇续写" 与 "分级智能润色"，打通从灵感到影视化改编的完整创作流。

一句话价值主张：让网文作者 10 分钟拿到可用于拍摄的剧本初稿，让编剧从手动改编中解放。

1.2 核心差异化

维度文枢现有竞品
长篇一致性上下文锚点系统，锁定角色/道具无记忆或简单向量检索
剧本改编专业心理外化策略 + 行业格式导出简单对话提取，无格式
创作流整合写-改-编一站式，无需跨工具功能割裂
文风定制用户上传作品学习专属风格仅预设模板

1.3 产品目标

  • V1.0：验证"小说转剧本"PMF，50%内测用户可直接使用生成剧本
  • V1.1：形成"写-改-编"最小闭环，完成完整用户链路
  • V2.0：通过版本快照、角色图谱、文风市场建立资产壁垒

2. 市场分析与竞品

2.1 市场机会

  • 网文市场规模：2025年中国网络文学市场规模超 400 亿元，作者群体超 2000 万
  • 短剧市场爆发：2025年短剧市场规模达 500 亿元，大量剧本需求
  • 影视改编需求：每年数百部网文出售影视版权，但专业改编人才稀缺
  • AI写作渗透率：约 30% 的网文作者尝试过 AI 辅助，但专业工具缺失

  3. 社区壁垒：作者角色库、世界观模板沉淀

3. 用户画像与场景

3.1 核心用户画像

画像A：网文大神/腰部作者

  • 年龄：25-35岁 · 特征：有完本作品，月收入 5000-50000
  • 痛点：作品售出影视版权但不会写剧本；长篇人设管理困难
  • 需求：一键剧本转换 + 一致性检查

画像B：短剧编剧

  • 年龄：22-30岁 · 特征：MCN/短剧公司签约，日产 3-5 集
  • 痛点：从网文/IP 中提炼剧本效率低；手动改格式耗时
  • 需求：快速转化 + 标准格式导出

画像C：新人作者

  • 年龄：18-28岁 · 特征：正在创作第一部作品，卡文频发
  • 痛点：不知如何推进剧情；人设经常矛盾
  • 需求：智能续写 + 设定管理

4.1 用户账户与项目管理

账户体系

  • 注册方式：邮箱 + 密码注册，注册后发送验证邮件，24 小时内点击激活链接完成验证；已验证邮箱方可正常使用 AI 功能
  • 登录方式：邮箱 + 密码登录；支持「忘记密码」通过邮件重置；预留第三方绑定扩展位（微信、GitHub）
  • 密码要求：8–32 位，至少包含字母和数字；后端 BCrypt(cost=12) 哈希存储，禁止明文
  • 双 Token 策略：Access Token（JWT，15 分钟有效）+ Refresh Token（随机字符串，30 天有效，存 Redis，支持主动吊销）；前端无感刷新
  • 密码重置：用户点击「忘记密码」→ 输入注册邮箱 → 系统发送重置链接（有效期 1 小时，含 UUID token）→ 点击链接进入重置页 → 设置新密码
  • 身份选择：首次登录引导选择身份（网文作者 / 短剧编剧 / 新人作者），影响功能入口顺序和引导文案，不限制任何功能使用
  • 配额显示：页面顶部全局展示「本月已用 / 总配额」进度条，悬停展示剩余字数、改编次数、套餐到期日

项目（作品）管理

  • 作品层级：项目 → 卷 → 章节，支持版本快照
  • 作品列表：卡片式展示，显示类型标签（颜色 + 文字双重标识）、总字数、最后修改时间；支持按修改时间 / 创建时间 / 字数排序
  • 快捷操作：继续写作 / 转剧本 / 重命名 / 复制 / 导出全书 / 删除（删除有二次确认）
  • 版本快照：每次 AI 大改前自动创建；全书替换操作前自动创建；用户可手动创建并备注；支持快照间 diff 对比；可按类型筛选（全部 / 手动 / AI改编前 / 润色前 / 替换前）；可恢复到任意版本。（DB 枚举值对照：手动 → manual，AI改编前 → auto_before_ai，润色前 → polish_accepted，全书替换前 → auto_before_replace；前端展示时做 i18n 映射，不直接显示英文枚举值。）

大纲面板

编辑器左侧面板，展示当前作品的完整卷章结构，支持以下操作：

  • 章节状态追踪：每章显示三种状态——待写（○）/ 写作中（→）/ 已完成（✓），状态可手动切换
  • 进度统计：底部实时显示「已完成章节数 / 总章节数」及进度百分比
  • 快速导航：点击任意章节直接跳转，不关闭面板
  • 章节管理：右键菜单支持编辑标题 / 在上方插入章节 / 删除章节

角色库面板

编辑器左侧面板，管理当前作品全部角色档案：

  • 角色信息：每个角色记录姓名、角色类型（主角/配角/反派/龙套）、外貌、性格、能力、说话风格、当前状态（位置/情绪/持有物）
  • 搜索与筛选：支持按角色名关键词搜索；支持按角色类型筛选
  • 锁定机制：锁定的角色档案在 AI 续写和改编时强制注入上下文，避免矛盾；未锁定角色按重要性评分动态选入
  • 自动更新：每章写完后，系统异步提取本章角色变化并更新档案（用户可手动修改）

全文搜索与替换

编辑器内置的全书级搜索与替换工具，触发方式为 Cmd+F（Mac）/ Ctrl+F（Windows），或命令面板搜索「全文搜索」。解决长篇创作中角色名全局修改、错别字批量纠正等高频刚需：

  • 全书搜索：在全部卷章范围内检索关键词，结果按章节分组展示，每条结果显示所在章节名、当章匹配数量及前后各 30 字上下文片段；搜索在后端执行，不受编辑器按章节加载限制
  • 快速跳转：点击任意搜索结果直接切换至对应章节并滚动高亮定位；支持「上一个（Shift+Enter）/ 下一个（Enter）」键盘导航，不需要关闭搜索面板
  • 精确替换：支持单条替换和全部替换两种模式；全部替换前展示匹配总数摘要供用户确认；替换操作自动创建版本快照（类型 auto_before_replace），支持一键撤销整次替换
  • 角色名联动替换：当替换内容命中角色库中某角色的姓名时，系统额外提示「"林天"在角色档案中存在，是否同步更新角色档案名称及别名？」；确认后角色档案自动更新，全库保持一致
  • 搜索选项：支持「区分大小写（默认关闭）」「全词匹配」两个可选开关，满足精确定位需求

写作统计看板

为有日更压力的职业作者提供创作数据可视化，量化写作成果并帮助建立写作习惯。入口位于首页顶部导航「统计」图标；编辑器左侧边栏「统」图标可快速查看当日概览而不离开写作界面：

  • 今日概览：实时统计当日手动输入字数与 AI 接受字数，显示合计及与每日目标的对比进度环；完成目标时触发正向反馈动效（彩带动画 + Toast 祝贺文案）
  • 连续日更：记录连续写作天数（当日净增字数 ≥ 用户设定的日目标下限即计一天），展示当前连续天数和历史最长记录；今日还未达标时，编辑器顶部展示轻提示「你已连续更新 N 天，今天还差 X 字」（可关闭）
  • 趋势图表：柱状图展示近 7 天 / 近 30 天每日字数，支持切换「手动输入 / AI 辅助 / 合计」三种维度；图表可下载为图片，用于社交媒体分享
  • 写作时段热力图：以「小时 × 星期几」维度的热力图展示用户活跃写作时段，帮助作者了解自身创作规律；数据来源为后端统计的 peak_hour 字段聚合
  • 作品进度统计：每部作品的总字数、已完成章节数 / 总章节数、近 30 天平均日更速度、按当前速度预计完结时间
  • 每日目标设置：默认 2000 字/日，可全局自定义；支持按单部作品设置不同目标（如更新中作品 3000 字、新作品 1000 字）；全局目标存入 users.daily_char_goal，作品目标存入 projects.daily_char_goal（值为 0 时沿用全局目标）
  • 月度摘要：每月 1 日在首页展示上月写作摘要卡片（总字数、写作天数、最高单日产量、最活跃作品），可保存图片分享；历史月度摘要可在统计页随时查看

内容导入

支持将已有作品导入文枢，解决存量作者的迁移需求：

  • 导入格式：支持 .txt（UTF-8）、.docx 两种格式，单文件最大 20MB（约 200 万字）
  • 章节自动切分：系统识别常见章节标题格式（第X章、Chapter X、数字编号等）自动切分，无法识别时以每 3000 字为单位切分，用户确认后正式导入
  • 粘贴导入：支持直接在新建作品页粘贴文本内容（上限 50 万字/次），快速进入校正和改编流程
  • 导入预览：导入前展示章节切分预览，用户可手动调整切分点，确认后才写入数据库
  • 元数据提取：导入完成后异步触发骨架分析（提取现有角色、世界观关键词），填充角色库和专有名词词典

专有名词 / 世界观词典面板

编辑器左侧面板（与角色库并列），管理作品中全部自创词汇，解决 AI 续写时专有名词混淆问题：

  • 词条类型：功法/技能、地名/场景、势力/门派、物品/道具、专有称谓，五类标签区分
  • 词条内容：每条记录名称、别名（如"苍龙诀"又称"苍诀"）、一句话解释、首次出现章节
  • AI 续写注入：词典中的词条在 AI 续写时强制注入上下文（与锁定角色档案享有同等优先级），防止 AI 写出"九天玄雷诀"/"玄天九雷诀"混用等错误
  • 自动提取：每章写完后，系统异步识别疑似专有名词（连续大写汉字组合、特殊词缀如"诀/功/域/界"等）提示用户确认添加到词典
  • 搜索与筛选：支持关键词搜索；支持按词条类型筛选
  • 容量限制：免费版 50 条 / 专业版 300 条 / 大师版 1000 条 / 团队版不限

数据管理与账户安全

  • 数据导出：用户可随时申请导出全部作品数据（含正文、角色档案、版本历史），系统异步打包为 .zip 文件，完成后 Toast 通知下载
  • 账号注销：软删除，注销后数据保留 30 天，期间可撤销；30 天后彻底清除
  • AI 训练授权：平台使用用户内容优化模型的授权可在账户设置中随时关闭，关闭后立即生效
账户安全机制
• 连续 5 次密码错误锁定账户 15 分钟（Redis 计数，key: login:fail:{email}）
• 异地登录发送邮件告警（IP 归属地变化超过 500km）
• Refresh Token 轮换机制：每次使用后颁发新 Token，旧 Token 立即失效
• 支持「登出所有设备」：Redis 删除该用户所有 Refresh Token

详细交互见《交互设计文档》第 0、1、2.4、2.5、13 章。

4.2 AI 写小说

骨架式生成

用户输入梗概（50–500字）并选择类型，AI 分三步生成故事骨架：①分析故事类型 → ②生成角色列表 → ③规划卷章结构。全程约 10–20 秒，界面实时显示当前步骤和进度。生成完成后用户可在预览界面编辑大纲树和角色档案，确认后才正式创建作品并进入编辑器。

分段续写

基于上下文锚点续写，支持 SSE 流式输出（字符逐步显示），用户可接受（融入正文）、重写（保持参数重新生成）或选择分支走向。续写结果自动创建版本快照后才融入正文。

互动卡点（分支建议）

在关键决策点提供 3 个不同风格的剧情走向供选择。

触发条件（满足任一即触发）：

  1. 章节字数达到 800 字，且当前段落末句含情绪高点关键词（爆发、决裂、死亡、表白、背叛等），底部出现「有新的走向建议」轻提示条，用户主动点击才展开，不自动弹窗
  2. 用户主动点击底部指令栏「分支」按钮
  3. 用户通过命令面板（Cmd+K）搜索「分支建议」
设计说明：写作者思考停顿超过 30 秒是常态，自动弹窗会打断深度写作状态，已去除「无操作自动触发」逻辑。分支建议改为完全由用户主动触发。

用户选择走向后，系统将走向简述注入续写 Prompt，确保 AI 续写内容与所选走向一致。支持「自定义走向」（用户手写走向描述后 AI 续写）和「重新生成」（刷新 3 个走向）。

上下文锚点系统

动态组装最小必要上下文包，保证长篇一致性。技术实现见《技术方案文档》。

4.3 AI 改小说

分级润色

级别操作示例
基础错别字、标点、的地得"他高兴的说" → "他高兴地说"
进阶句式优化、消除翻译腔"他进行了思考" → "他思索片刻"
风格改变叙事节奏/氛围舒缓 → 紧张，平实 → 古风

风格重塑与文风定制

风格重塑：选中文字后，可选择将其改写为「古典/古风」「快节奏/紧张」「舒缓/治愈」「幽默/轻松」四种预设风格，保留所有情节内容。

文风定制（专业版）：用户上传自己的代表作片段（最多 3 段，每段 300–1000 字），AI 分析提取写作风格特征（句式偏好/词汇风格/叙事节奏），在续写和润色时以此为参考范例。

⚠️ 重要说明：文风定制通过分析样本片段、注入 Few-shot 示例的方式实现，不需要训练模型、不收集模型训练数据，设置后即刻生效。

全卷一致性审查

扫描全书角色外貌矛盾、能力矛盾、时间线冲突、地理场景矛盾等问题。

审查报告交付形式：审查完成后在右侧面板展示矛盾报告，每条记录包含：

  • 矛盾类型（外貌 / 能力 / 时间线 / 场景）
  • 涉及角色名称和所在章节位置
  • 前后矛盾的具体描述（引用原文）
  • 建议修改方向
  • 操作按钮：「跳转对应章节」「标记已处理」「忽略此条」

报告可导出为 .txt 文件，支持在其他工具中查阅。

一致性审查配额限制

全卷一致性审查涉及多次 LLM 调用，按套餐设定字数上限和月次数限制，防止成本失控。

套餐单次最大审查字数每月次数超限行为
免费版2 万字（约 5 章）1 次 / 月弹出升级引导弹窗，不可继续
专业版50 万字5 次 / 月Toast 提示「本月审查次数已用完，下月重置」
大师版200 万字20 次 / 月Toast 提示，不强制拦截
团队版不限不限—

超出单次字数上限时，系统提示「当前作品共 XX 万字，超出 XX 套餐上限，可选择：① 升级套餐 ② 仅审查指定章节范围（最近 N 章）」，支持用户灵活选择审查范围。

成本控制策略：全卷一致性审查采用 DeepSeek Chat 模型（而非 Claude）执行文本分块审查，在保证质量的前提下将单次 API 调用成本降低约 10 倍。仅在发现疑似矛盾需要深度推理验证时，才升级调用 Claude 进行二次确认（每次审查最多触发 5 次 Claude 调用）。此策略对用户完全透明，不影响审查结果质量。

4.4 小说转剧本

核心流程

选择章节范围 → 选择心理外化策略 → AI 改编工作台（原文 / 剧本 / AI建议三栏）→ 用户逐场确认与编辑 → 格式导出。交互细节见《交互设计文档》第 7 章。

改编进度反馈

剧本改编为异步任务（约 30–90 秒），改编期间工作台中间列实时显示进度：「正在改编第 N 场 / 共 M 场」和进度百分比。已完成的场景可在生成过程中提前查看和编辑，无需等待全部完成。用户可随时点击「取消改编」中止任务。

工作台功能

  • 三栏联动：点击原文段落，剧本对应场景自动滚动高亮
  • 策略选择：每个场景单独选择心理外化策略（动作 / 对白 / 独白OS / 跳过），AI 提供推荐方案；也可「全部用推荐」一次性批量应用
  • 保存草稿：工作台编辑中途可手动保存草稿，下次打开同一作品时恢复进度
  • 格式校验：导出前自动检查格式合规性（场景标题格式、心理描写是否外化等），以提醒形式展示，不阻止导出
  • 分集管理：改编完成后支持将场景按集归组（短剧场景通常每集 3-6 场），用户可手动拖拽场景到对应集，或使用「AI 自动分集」（系统按情节节奏建议分集方案）；支持按集独立导出，满足短剧编剧日产多集的工作流
  • 多设备并发保护：同一剧本草稿在多个标签页或多台设备同时打开时，采用乐观锁版本号机制；当保存操作发生版本冲突时，系统弹出对比对话框，用户可选择「保留我的修改」或「使用最新版本」，不会静默覆盖任何内容

导出格式

中文影视标准格式（.docx）、Final Draft (.fdx)、简单分镜表，三种格式可在导出预览界面实时切换预览后再下载。导出文件自动添加「AI辅助生成」页脚标注。

新用户体验额度策略

  • 额度提示：体验额度耗尽时，系统展示「体验额度已用完，订阅专业版继续创作」引导，而非冷冰冰地禁用功能
  • 邀请加赠：每成功邀请一位新用户注册，双方各额外获得 10,000 字体验额度（上限 50,000 字）

  • 角色权限：管理员（可查看/编辑全部作品）/ 主编（可审核成员提交）/ 写手（只能编辑被分配的作品），三级权限
  • 作品协同：同一作品可指定多名写手分章写作，各章节独立锁定，防止同时编辑冲突

7. 合规与伦理

7.1 版权与免责

  • 用户拥有其输入和生成内容的所有权
  • 平台可使用用户内容优化模型（可关闭）
  • 导出剧本自动添加“AI辅助生成”标注
  • 禁止生成违法违规、侵权、色情内容

7.2 内容安全

输入/输出实时敏感词过滤，流式替换/阻断，举报+抽检。

7.3 数据隐私

作品默认私有，分享链接24h失效，用户可随时导出数据，注销后30天内彻底删除。
